🔢 Cálculo de Tasas (λ y μ)

Conversión de Tiempos a Tasas Para distribuciones exponenciales: λ = 1 / (Tiempo entre llegadas) [clientes/minuto] μ = 1 / (Tiempo de servicio) [clientes/minuto] Cálculos para cada tipo de usuario: 🟢 RÁPIDO: λ = 1/3 = 0.333 clientes/min μ = 1/1 = 1.000 clientes/min ρ = λ/μ = 0.333 (Sistema estable ✓) 🔵 NORMAL: λ = 1/3 = 0.333 clientes/min μ = 1/3 = 0.333 clientes/min ρ = λ/μ = 1.000 (Sistema crítico ⚠️) 🟠 LENTO: λ = 1/5 = 0.200 clientes/min μ = 1/4 = 0.250 clientes/min ρ = λ/μ = 0.800 (Sistema estable ✓) 🔴 MUY LENTO: λ = 1/7 = 0.143 clientes/min μ = 1/6 = 0.167 clientes/min ρ = λ/μ = 0.857 (Sistema estable ✓) Nota: ρ (rho) es el factor de utilización. Si ρ ≥ 1, el sistema es inestable.
⚠️ Observación Crítica:

Los usuarios NORMALES tienen un factor de utilización ρ = 1.0, lo que indica que el sistema está en el límite de estabilidad. Esto puede generar colas infinitas en teoría. Los usuarios LENTOS y MUY LENTOS representan el 55% del total, lo que puede generar congestión significativa.

📚 Fundamentos Teóricos: Modelo M/M/1

🎓 ¿Qué es el Modelo M/M/1?

El modelo M/M/1 es un sistema de colas con las siguientes características:

  • M (primera): Llegadas siguen un proceso de Poisson (tiempos exponenciales)
  • M (segunda): Tiempos de servicio siguen distribución exponencial
  • 1: Un solo servidor
  • Disciplina: FIFO (First In, First Out)
  • Capacidad: Infinita (sin límite de cola)

📐 Fórmulas Clave del Modelo M/M/1

Métricas de Desempeño M/M/1 Parámetros básicos: λ = Tasa de llegadas (clientes/unidad de tiempo) μ = Tasa de servicio (clientes/unidad de tiempo) ρ = λ/μ = Factor de utilización (debe ser < 1 para estabilidad) Métricas principales: 1. Número promedio de clientes en el sistema (L): L = ρ / (1 - ρ) = λ / (μ - λ) 2. Número promedio de clientes en la cola (Lq): Lq = ρ² / (1 - ρ) = λ² / (μ(μ - λ)) 3. Tiempo promedio en el sistema (W): W = 1 / (μ - λ) 4. Tiempo promedio en la cola (Wq): Wq = ρ / (μ - λ) = λ / (μ(μ - λ)) 5. Probabilidad de que el sistema esté vacío (P₀): P₀ = 1 - ρ 6. Probabilidad de n clientes en el sistema (Pn): Pn = (1 - ρ) × ρⁿ Fórmulas de Little: L = λ × W Lq = λ × Wq

🎯 Ejemplo de Cálculo: Usuario Rápido

📊 Cálculo Teórico para Usuario Rápido
Datos: λ = 1/3 = 0.333 clientes/min μ = 1/1 = 1.000 clientes/min ρ = 0.333/1.000 = 0.333 Cálculos: L = 0.333 / (1 - 0.333) = 0.333 / 0.667 = 0.50 clientes Lq = 0.333² / (1 - 0.333) = 0.111 / 0.667 = 0.17 clientes W = 1 / (1.000 - 0.333) = 1 / 0.667 = 1.50 minutos Wq = 0.333 / (1.000 - 0.333) = 0.333 / 0.667 = 0.50 minutos P₀ = 1 - 0.333 = 0.667 (66.7% del tiempo el cajero está vacío) Interpretación: - En promedio hay 0.5 clientes en el sistema - El tiempo promedio total es 1.5 minutos - El cajero está ocupado solo el 33.3% del tiempo - Sistema muy eficiente para usuarios rápidos ✓

def determinar_estado_estable(datos, ventana=50): """ Determina el punto donde el sistema alcanza estado estable usando promedios móviles """ promedios_moviles = [] for i in range(len(datos) - ventana): promedio = np.mean(datos[i:i+ventana]) promedios_moviles.append(promedio) # Detectar estabilización (varianza baja) varianzas = [] for i in range(len(promedios_moviles) - ventana): var = np.var(promedios_moviles[i:i+ventana]) varianzas.append(var) # Punto de corte: donde varianza se estabiliza punto_corte = np.argmin(varianzas) + ventana return punto_corte, promedios_moviles
